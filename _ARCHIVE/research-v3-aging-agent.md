# 高齢化社会 × AIエージェント：課題調査レポート v3
調査日: 2026-03-07

## 調査方針
チャットBot（情報提供のみ）ではなく、AIエージェントが複数ステップのタスクを自律実行して問題を解決するアイデアに絞って調査した。

---

## アイデア一覧（8案）

---

### アイデア1: ケアプラン素案 自動生成エージェント
**ターゲット:** ケアマネージャー（居宅介護支援事業所）

#### ペインの深刻度
- ケアマネの業務時間の約50%が事務・記録業務（訪問と書類作業が半々）。**Confirmed**（厚生労働省・居宅介護支援事業所実態調査）
- 「記録する書式が多く手間がかかる」と回答したケアマネが**約70%**。**Confirmed**（mhlw.go.jp調査）
- サービス担当者会議の議事録作成が従来3時間 → AI導入後30〜45分に短縮できる事例あり（noman社データ）。**Confirmed**
- 1名あたりの記録作成業務工数が43%削減、残業時間が30時間→6時間になった施設事例。**Confirmed**（noman社）
- 2024年度介護報酬改定で担当件数の逓減制が39件→44件に緩和されたが、ケア質低下を懸念する声が多数。**Confirmed**（care-news.jp）

#### AIエージェントが代行するステップ
1. アセスメント情報（利用者の状態・ニーズ）をヒアリング形式で構造化入力
2. アセスメントデータから課題を抽出・整理
3. ケアプラン第1表（総合的な援助の方針）・第2表（目標・サービス内容）の素案を自動生成
4. 必要なサービス種別・提供事業者候補をリストアップ
5. サービス担当者会議の議事録フォーマット（介護給付費算定上必須）を自動整形して出力

#### 7時間プロトタイプ範囲
- アセスメントJSON → ケアプラン第2表テキスト素案生成（LLM + プロンプトエンジニアリング）
- フォームUI（Next.js or Lovable）でデモ可能

#### 競合・既存
- noman（議事録専門）、ミルモレコーダー（音声文字起こし）
- 厚労省のAIケアプラン実証は第3フェーズで研究段階（2023〜）
- ワイズマン「AIケアプラン」は大企業向け・高額
- **差別化余地:** アセスメントから提案まで一気通貫でエージェント的に動作するSaaS型

### アイデア2: 介護保険 更新申請 書類準備エージェント
**ターゲット:** 在宅で親の介護をするワーキングケアラー

#### ペインの深刻度
- 介護離職者は2024年に約9.3万人（2000年比2.4倍）。**Confirmed**（総務省統計局 就業構造基本調査）
- ワーキングケアラー365万人のうち、週4日以上介護に携わる人が45%。**Confirmed**（2024年調査）
- 1日平均3時間の介護に加え、各種手続きが仕事の隙間に発生する。**Confirmed**
- 要介護認定の有効期間は1〜3年で、更新ごとに申請が必要。申請から認定まで約1ヶ月かかる。**Confirmed**（各自治体ウェブサイト）
- デジタル手続きを知らない・使えない高齢者が多く、家族が代行するケースが大多数。**Confirmed**（総務省情報通信白書）

#### AIエージェントが代行するステップ
1. 現在の認定状況・有効期限を確認し、更新申請のタイミングをアラート
2. 居住地の市区町村を特定し、必要書類・申請先窓口・提出方法を自動調査
3. 申請書の各欄を質問形式で埋めていき、草案を生成
4. 主治医意見書の依頼状（医師へのレター）を自動起草
5. 申請後の調査員訪問に向けた「伝えるべきポイントリスト」を生成

#### 7時間プロトタイプ範囲
- チャット形式でヒアリング → 申請書草案PDF出力 + 医師への依頼文生成（LLM + PDF生成）

#### 競合・既存
- 行政書士サービス（有料・手書き中心）
- みんなの介護、LIFULL介護（情報提供のみ、エージェント機能なし）
- **差別化余地:** 申請書自動記入 + 市区町村別の手続きフローを自動調査するエージェント

### アイデア3: 複数介護施設 一括入所申し込みエージェント
**ターゲット:** 特養・有料老人ホームへの入所を急ぐ家族

#### ペインの深刻度
- 特養の入所申し込みから入所まで「1年以上かかった」が35.8%。待機期間が長期化。**Confirmed**（介護施設調査）
- 複数施設への申し込みは各施設ごとに書類を別途準備・提出する必要がある。**Confirmed**（各施設案内）
- 希望施設が多いほど「書類の重複準備」の手間が線形に増加する。**Likely**
- 家族が施設見学・書類準備・郵送・フォローアップを全て行う。1施設あたり半日〜1日の工数。**Likely**

#### AIエージェントが代行するステップ
1. 条件（エリア、費用上限、要介護度、特徴）を入力して候補施設リストを自動生成
2. 各施設の入所申込書フォーマットを収集・統合
3. 1回の情報入力から、各施設形式に合わせた申込書を一括生成
4. 施設への問い合わせメール（見学希望・申込書送付）を自動起草・送信
5. 各施設のステータス（待機順位確認、連絡要否）をトラッキング

#### 7時間プロトタイプ範囲
- 利用者情報を1回入力 → 複数施設申込書（PDF）を一括生成するデモ
- 施設検索は介護サービス情報公表システムAPIまたはスクレイピング

#### 競合・既存
- LIFULL介護（施設検索のみ、申し込み代行なし）
- みんなの介護（情報提供のみ）
- **差別化余地:** 申込書の一括生成・送信というタスク代行部分に特化

### アイデア4: 遠距離介護 状況把握 & 手配自動化エージェント
**ターゲット:** 親と離れて暮らすビジネスケアラー

#### ペインの深刻度
- 別居介護の割合が2013年の9.6%→2019年の13.6%に増加、絶対数では数百万人規模。**Confirmed**
- ビジネスケアラー318万人（2025年推計）、経済損失9兆円。**Confirmed**（経産省資料）
- 「帰省のたびに複数機関との調整・手続き・確認」に追われ、仕事への影響が深刻。**Confirmed**
- 遠距離介護者は移動コストに加え「情報の非対称性」（現地状況を把握できない）が最大の課題。**Confirmed**

#### AIエージェントが代行するステップ
1. ケアマネ・ヘルパー・かかりつけ医からの報告メール・電話メモを自動集約・要約
2. 週次サマリー（状態変化・課題・対応が必要な事項）を自動生成して家族にメール
3. 必要な対応（次回診察の予約、サービス変更申請など）をエージェントがリストアップ
4. 「次の帰省でやること」チェックリストを自動生成
5. 緊急時（入院等）に備えた連絡先・手続きリストを状況に応じて自動更新

#### 7時間プロトタイプ範囲
- テキスト入力（ケアマネからの連絡文）→ 要約 + アクションアイテム抽出 + 家族への週次レポート生成
- デモとしてSimulated feedを使い一連のフローを見せる

#### 競合・既存
- 見守りカメラ・IoTデバイス（センサーのみ、コンテキスト統合なし）
- ちかく（NTTドコモ）：見守り特化
- **差別化余地:** 複数情報源（ケアマネ連絡 + 薬局 + 病院）を統合してアクション提案するエージェント

### アイデア5: ケアマネ向け サービス担当者会議 議事録自動生成エージェント
**ターゲット:** ケアマネージャー

#### ペインの深刻度
- サービス担当者会議の議事録は介護報酬算定の要件として義務提出。**Confirmed**（介護保険法）
- 従来の議事録作成に3時間かかっていたものが、AI導入後30〜45分に短縮された事例。**Confirmed**（noman社事例）
- 残業時間が月30時間→6時間に削減された施設管理者の事例。**Confirmed**（noman社）

#### AIエージェントが代行するステップ
1. 音声録音をリアルタイム文字起こし
2. 参加者・発言・決定事項を自動分類
3. 介護給付費算定要件を満たすフォーマット（サービス担当者会議の要点）に自動整形
4. ケアプランの変更点を抽出し、次回モニタリング課題を自動リストアップ
5. 各サービス提供事業者への通知メール草案を自動生成

#### 7時間プロトタイプ範囲
- 音声または文字起こしテキスト入力 → 議事録フォーマット出力
- noman等の競合があるが、「議事録→ケアプラン更新提案→通知メール」まで繋げる点で差別化

#### 競合・既存
- **noman（最強の競合）**: 既に40〜70%業務削減実績あり
- ミルモレコーダー（音声→記録）
- **重要注記:** このアイデアはnomanと直接競合。差別化がない場合は避けるべき。

### アイデア6: 高齢者の外来受診 代行予約・情報整理エージェント
**ターゲット:** デジタルが苦手な高齢者本人 or 代わりに手続きする家族

#### ペインの深刻度
- 70歳以上のスマートフォン利用率は40.8%（60〜69歳は73.4%）。**Confirmed**（総務省情報通信白書）
- 複数科受診・多数の病院を抱えるポリファーマシー患者では、病院間の情報連携が欠如。**Confirmed**
- オンライン予約を「知らない」「使い方がわからない」高齢者が多数。**Confirmed**
- 家族が「代わりに調べて予約する」タスクが無限に発生している。**Likely**

#### AIエージェントが代行するステップ
1. 「次の〇〇科の受診をお願いしたい」と話すだけで、かかりつけ医のオンライン予約ページにアクセス
2. 希望の日程・医師・診療科を選択して予約を完了
3. 前回受診の内容・薬の変更点をまとめた「受診メモ」を自動生成
4. 複数病院の診察日程を統合カレンダーで管理し、重複や飲み合わせ問題をチェック
5. 診察後の薬の飲み方を音声で説明するリマインダーをセット

#### 7時間プロトタイプ範囲
- 音声入力 → 予約フォーム自動入力 or メール起草 + 受診メモ生成
- デモ環境での予約エージェントフロー（Playwright/Puppeteerによるブラウザ自動操作）

#### 競合・既存
- 病院の予約システム（自分で操作必要）
- ウェルヴィル「LIFE TALK 病院受付」（受付対話AI、予約代行ではない）
- **差別化余地:** 音声で依頼 → 実際に複数病院のウェブ予約を完了させるエンドツーエンドのエージェント

### アイデア7: 介護開始時 行政手続き ワンストップエージェント
**ターゲット:** 突然の親の入院・介護状態悪化に直面した家族

#### ペインの深刻度
- 要介護認定申請 → 認定まで最大30日、認定後にケアマネ選定 → ケアプラン作成 → サービス開始まで数週間〜数ヶ月の手間がある。**Confirmed**（各自治体サイト）
- 「何から始めればいいかわからない」が介護開始時の最大の課題。**Confirmed**（複数調査）
- 介護保険申請・高額療養費申請・障害者手帳・成年後見制度など、複数の手続きが並行発生する。**Confirmed**
- 働きながら介護を経験した人でも3割以上が「介護休業」を知らなかった。**Confirmed**（LIFULL介護 2024調査）

#### AIエージェントが代行するステップ
1. 現在の状況（親の年齢・疾患・居住地・家族構成）をヒアリング
2. 利用できる制度・補助金・サービスをリストアップして優先順位を付ける
3. 各手続きの申請書を居住地に合わせて自動で収集・草案作成
4. 手続きの締め切り・順番・依存関係を整理したロードマップを生成
5. ケアマネ・地域包括支援センターへの初回連絡メールを自動起草

#### 7時間プロトタイプ範囲
- 状況ヒアリング（フォーム or チャット）→ 手続きロードマップ生成 + 申請書草案（主要2〜3点）の自動生成
- 居住地で変わる制度差分を LLM + 構造化データで対応

#### 競合・既存
- 地域包括支援センター（人手不足で相談待ちが発生）
- 行政書士（有料）
- **差別化余地:** 「今すぐ無料で、具体的な書類まで出してくれる」エージェント。人手不足の相談センターの代替として機能

### アイデア8: 終活・エンディングノート 自動整理エージェント
**ターゲット:** 終活を始めたい高齢者本人、または親の終活を支援したい子世代

#### ペインの深刻度
- 少子高齢化で終活市場は「絶対的にマーケットが大きくなっている領域」。**Confirmed**（VC・起業家調査）
- エンディングノートを書いていても「情報が散在」「家族が見つけられない」「更新されない」問題が深刻。**Likely**
- 遺言書・相続手続き・デジタル遺品整理の複雑さが家族の大きな負担になっている。**Confirmed**（相続手続き白書等）
- 日本初の「AI連動終活手帳（スマート終活手帳）」が2026年版として予約開始されるなど市場が動いている。**Confirmed**（PR TIMES 2025）

#### AIエージェントが代行するステップ
1. 対話形式で必要情報（資産・希望・家族構成・医療意思）をヒアリング
2. エンディングノートの各項目を自動整理・記入
3. 関連する手続き（遺言書の必要性診断、公証人への連絡方法）を自動提案
4. 金融口座・デジタルアカウントのリストを整理するためのチェックリストを生成
5. 家族への「伝言メッセージ」文案を起草

#### 7時間プロトタイプ範囲
- チャット形式でヒアリング → 構造化されたエンディングノートPDF生成 + 必要手続きリスト
- MUFG「わが家ノート」、スマート終活手帳との差別化は「エージェントが能動的に情報を引き出す」点

#### 競合・既存
- MUFG「わが家ノート by MUFG」（フォーム形式、エージェントではない）
- AI VOLT「TOKIAI」（デジタルアバター生成）
- スマート終活手帳（フィジタル商品）
- **差別化余地:** 「聞かれれば答えるのではなく、エージェントが足りない情報を能動的に収集し、法的に必要な手続きまで繋げる」点

## 総評と推奨ランキング

| ランク | アイデア | 理由 |
|--------|----------|------|
| ★★★ | アイデア2（介護保険申請） | ペイン深刻・競合弱・7時間で完成度高いプロトタイプ可能 |
| ★★★ | アイデア7（介護開始時ワンストップ） | ペイン最深刻・「何から始めればいい」という全介護者共通の課題 |
| ★★☆ | アイデア3（施設入所一括申込） | 書類一括生成のエージェント価値が明確・競合弱 |
| ★★☆ | アイデア4（遠距離介護状況把握） | 市場大・エージェント性高い・デモが映えるが7時間では浅くなりがち |
| ★★☆ | アイデア1（ケアプラン素案生成） | ニーズ確実・競合多いが実務ユーザーへのインパクト大 |
| ★☆☆ | アイデア6（受診予約代行） | 技術的に可能だが医療機関側の協力が必要でプロトタイプが難しい |
| ★☆☆ | アイデア8（終活エージェント） | 市場は大きいが競合増加中・ハッカソンで差別化が難しい |
| △ | アイデア5（会議議事録） | nomanと直接競合のため差別化困難 |
