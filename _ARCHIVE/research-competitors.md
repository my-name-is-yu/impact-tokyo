# ポリファーマシー 競合調査レポート
*作成: 2026-03-07 | IMPACT TOKYO 2026 ハッカソン向け*

---

## 背景・問題規模

| 指標 | データ |
|------|--------|
| 75歳以上の在宅医療患者でポリファーマシー（5剤以上）の割合 | **70%**（2015・2019ともに不変） |
| 高齢者の家庭医療患者のポリファーマシー有病率 | **50%以上** |
| 副作用リスク：単剤→多剤（ポリファーマシー）の相対リスク | **4.3倍** |
| 在宅高齢者のPIM（不適切処方）保有率 | **40.4%** |

## 3. 学術・行政の取り組み

### 厚生労働省の指針・施策

| 取り組み | 内容 |
|----------|------|
| 「高齢者の医薬品適正使用の指針（総論編）」（2018年） | ポリファーマシーの定義・対策プロセスを国が初めて明文化 |
| 「病院における高齢者のポリファーマシー対策の始め方と進め方」（2021年、2024年7月改訂） | 病院での実践的な減薬プロセスを提示 |
| 「地域における高齢者のポリファーマシー対策の始め方と進め方」（2024年） | 地域包括ケアへの展開を追加 |
| 服用薬剤調整支援料（診療報酬） | 薬剤師が減薬提案・実施した場合に診療報酬算定可。2024年度改定でカンファレンス要件を強化 |
| 電子処方箋（2023年〜） | 多施設間の重複投薬・禁忌チェックを自動化 |

### 国際基準・ツール

| 基準 | 内容 |
|------|------|
| STOPP/START criteria（欧州、v3 2023年） | 高齢者への不適切処方・過少処方を体系的に判定するチェックリスト（v3で項目数が大幅増加） |
| STOPP-J（日本版、2015年） | 日本の診療慣行に合わせて修正したPIM判定基準。在宅医療患者での予後との関連が実証済み |
| Beers criteria（米国） | 米国老年医学会による不適切薬剤リスト |

### 学術的ギャップ

- STOPP-J適用時、在宅高齢者の40.4%がPIMを保有しているが、**日常診療でSTOPP-Jが使われているケースは少ない**
- STOPP/STARTは項目数が多く（v3でさらに増加）、医師・薬剤師が手動で適用するには時間コストが高い

## 5. ギャップ分析：まだ解決されていない課題

### Gap 1: 多科受診データの統合が実用レベルで存在しない
- **問題:** 日本の高齢者は平均3〜4科の医療機関を受診し、それぞれで薬が処方される。電子処方箋はこれを解決するはずだが、**医療機関の導入率は10%未満**（2025年3月時点）
- **現状の穴:** 複数のクリニックをまたいだ処方情報を、患者自身も・薬剤師も・医師も「リアルタイムで全体像を把握する手段」が事実上ない
- 信頼度: **Confirmed**

### Gap 2: 患者・家族が「今の薬が適切か」を能動的に確認できるツールがない
- **問題:** 既存のお薬手帳アプリは「記録する」だけ。飲み合わせチェックは患者自身が入力する手間があり、結果の解釈も素人には難しい
- **現状の穴:** 「この組み合わせは問題ありますか？」を患者・家族が気軽に聞けるAIアシスタントが国内に存在しない
- 信頼度: **Confirmed**

### Gap 3: STOPP-J / STOPP/START の臨床実装が進んでいない
- **問題:** 不適切処方の判定基準（STOPP-J等）は学術的に確立されているが、項目数が多く（v3でさらに増加）、医師・薬剤師が日常業務でスクリーニングに使うには時間コストが高すぎる
- **現状の穴:** これをAIで自動スクリーニングするツールが日本国内に存在しない
- 信頼度: **Confirmed**（学術論文・実態調査より）

### Gap 4: 介護者・家族への情報共有・アクションループがない
- **問題:** 在宅高齢者の50%以上がポリファーマシー状態。家族が「何の薬を飲んでいるか・副作用サインは何か」を理解・把握する支援ツールがない
- **現状の穴:** Medisafe（海外）はMedfriend機能で一部対応しているが、日本語・日本の医薬品データベース対応なし
- 信頼度: **Confirmed**

### Gap 5: 薬剤師の介入提案を医師にフィードバックするループの非効率性
- **問題:** 薬剤師が減薬提案をしても、医師への疑義照会・フィードバックは紙・電話・FAXが主流。診療報酬（服用薬剤調整支援料）の算定率も低い
- **現状の穴:** 薬剤師の処方見直し提案を構造化し、医師にデジタルで届ける仕組みがない
- 信頼度: **Likely**（診療報酬改定資料・m3.com記事より）

### Gap 6: 高齢者本人のデジタルリテラシー問題
- **問題:** 既存アプリは高齢者自身には使いにくい。操作が複雑・画面が見にくい等
- **現状の穴:** 高齢者が自分で操作しなくてよい仕組み（家族・介護者経由、音声操作、LINEベース等）の設計が不足
- 信頼度: **Confirmed**

## 6. 差別化の機会：医学生×エンジニアチームだからこそ狙える隙間

### 狙い目ポジション: 「患者・家族向けAIポリファーマシー副操縦士」

既存ツールが医師向け・薬剤師向けに偏る中、**患者と家族が使える**、**AIが処方の全体像を解釈してくれる**プロダクトは空白地帯。

#### 具体的な差別化アイデア

| アイデア | 根拠となるギャップ | 医学生の強み | エンジニアの強み |
|----------|-----------------|------------|----------------|
| お薬情報を写真/QRで取り込み、STOPP-J基準でAIがリスク判定 → 家族にわかりやすく説明 | Gap 2 + Gap 3 | STOPP-J・薬理学知識でルール設計 | OCR+LLMパイプライン実装 |
| 複数の医療機関からの処方を統合管理し、重複・相互作用を患者目線で可視化 | Gap 1 + Gap 2 | 多科受診の臨床知識 | データ統合・UI設計 |
| 介護者・家族向けLINEボット：「この薬の副作用で注意すべき症状は？」「減薬できないか薬剤師に聞くには？」 | Gap 4 + Gap 6 | 医学的に正確な回答設計 | LINEボット実装 |
| 薬剤師向け: STOPP-J自動スクリーニング + 医師への疑義照会文書自動生成 | Gap 3 + Gap 5 | 疑義照会の臨床ルール知識 | LLM fine-tuning / prompt engineering |

#### 推奨アプローチ（ハッカソン7時間向け）

**「高齢者の家族向け × 薬の全体像チェックAI」**

1. 複数の薬袋をスマホカメラで撮影 or 処方箋QRを読み込み
2. 薬剤名・用量をOCRで抽出
3. LLMがSTOPP-J + 主要相互作用データベースで評価
4. 家族にわかりやすいレポートを生成（「この薬の組み合わせに注意点があります。薬剤師に確認してみましょう」）
5. 必要に応じて薬剤師・医師への相談テンプレートを自動生成

**競合との差別化:**
- 薬剤師向け（カケハシ）でも医師向け（FeelBetter）でもなく、**家族・患者向け**
- お薬手帳アプリと異なり**AIが判断・解釈**を提供（記録だけでなく意味を伝える）
- 日本のSTOPP-J・日本語薬剤データベースに特化
- 医師・薬剤師へのブリッジ（「こう質問してください」のスクリプト付き）
